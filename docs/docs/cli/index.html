CLI Tool

Interactive command-line interface for creating, managing, and testing AgentSea AI agents.

Built for Developer Productivity

✅ Interactive setup wizard

✅ Real-time chat interface

✅ Agent management (CRUD operations)

✅ Model management (pull, list, info)

✅ Configuration persistence

✅ Voice support

Installation

Install the AgentSea CLI globally with npm:

bash 
npm install -g @lov3kaizen/agentsea-cli
pnpm add -g @lov3kaizen/agentsea-cli
yarn global add @lov3kaizen/agentsea-cli
agentsea --version

Quick Start

Initialize AgentSea with the interactive setup wizard:

bash 
agentsea init
# The wizard will guide you through:
# 1. Choose your provider (Anthropic, OpenAI, Ollama, etc.)
# 2. Enter API key (if cloud provider)
# 3. Select default model
# 4. Configure voice providers (optional)
# 5. Save configuration

Configuration

Configuration is stored in:

  • macOS: ~/Library/Preferences/agentsea-cli
  • Linux: ~/.config/agentsea-cli
  • Windows: %APPDATA%/agentsea-cli

agentsea agent

Manage your agents - create, list, get, delete, run.

Create an Agent

bash 
agentsea agent create
agentsea agent create my-assistant   --provider anthropic   --model claude-sonnet-4-20250514   --system-prompt "You are a helpful coding assistant"   --temperature 0.7   --max-tokens 2048
agentsea agent create voice-assistant   --provider ollama   --model llama3.2   --voice-stt openai-whisper   --voice-tts openai-tts

List Agents

bash 
agentsea agent list
Available Agents:
================
my-assistant (default)
  Provider: anthropic
  Model: claude-sonnet-4-20250514
voice-assistant
  Provider: ollama
  Model: llama3.2
  Voice: OpenAI Whisper (STT) + OpenAI TTS (TTS)

  Model: gpt-4-turbo-preview

Run an Agent

bash 
agentsea agent run my-assistant "What is 42 * 58?"
agentsea agent run voice-assistant   --voice-input ./question.mp3   --voice-output ./response.mp3
agentsea agent run my-assistant "Explain async/await" {'>'} explanation.txt

Other Agent Commands

bash 
agentsea agent get my-assistant
agentsea agent delete my-assistant
agentsea agent default my-assistant

Common Workflows

1. Quick Local Setup

bash 
curl -fsSL https://ollama.com/install.sh | sh
agentsea init
# Select: Ollama (Local)
agentsea model pull llama3.2
agentsea chat
No API keys needed!
Complete privacy!
Zero costs!

2. Create Voice-Enabled Assistant

bash 
agentsea agent create voice-bot   --provider anthropic   --model claude-sonnet-4-20250514   --voice-stt openai-whisper   --voice-tts openai-tts   --system-prompt "You are a voice assistant. Keep responses concise."
agentsea chat --agent voice-bot --voice
agentsea agent run voice-bot   --voice-input question.mp3   --voice-output answer.mp3

3. Multi-Agent Setup

bash 
agentsea agent create researcher   --model claude-sonnet-4-20250514   --system-prompt "Research and gather information"
agentsea agent create writer   --model gpt-4-turbo-preview   --system-prompt "Write clear, engaging content"
agentsea agent create coder   --provider ollama   --model codellama   --system-prompt "Write clean, efficient code"
agentsea agent run researcher "Latest AI trends"
agentsea agent run writer "Blog post about AI agents"
agentsea agent run coder "Python function for data validation"

json

      "voice": "nova"

}

Tips & Best Practices

Use Specialized Agents

Create different agents for different tasks: code review, writing, data analysis, etc. Each can have optimized prompts and models.

Mix Cloud & Local

Use local models for development and simple tasks, cloud models for complex production work. Best of both worlds.

Keep Keys Secure

The CLI stores API keys securely in your system config. Never commit them to version control.

Voice Mode Tips

Keep voice assistant prompts concise. Add "Respond in 2 sentences max" for better voice UX. Users don't want to listen to long responses.
